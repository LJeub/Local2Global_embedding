Run training example.

By default this function writes results to the current working directory. To override this use the output argument.

Parameters:

  • name ‚Äì Name of data set to load (one of {'Cora', 'PubMed', 'AMZ_computers', 'AMZ_photo'})

  • data_root ‚Äì Directory to use for downloaded data

  • no_features ‚Äì If True, discard features and use node identity.

  • model ‚Äì embedding model type (one of {‚ÄòVGAE‚Äô, ‚ÄòGAE‚Äô, ‚ÄòDGI‚Äô})

  • num_epochs ‚Äì Number of training epochs

  • patience ‚Äì Patience for early stopping

  • runs ‚Äì Number of training runs (keep best result)

  • dims ‚Äì list of embedding dimensions (default: [2])

  • hidden_multiplier ‚Äì Hidden dimension is hidden_multiplier * dim

  • target_patch_degree ‚Äì Target patch degree for resistance sparsification.

  • min_overlap ‚Äì Minimum target patch overlap (default: max(dims) + 1)

  • target_overlap ‚Äì Target patch overlap (default: 2 * max(dims))

  • gamma ‚Äì Value of ‚Äògamma‚Äô for RMST sparsification

  • sparsify ‚Äì Sparsification method to use (one of {'resistance', 'none', 'rmst'})

  • train_directed ‚Äì Use the orignal directed network (only relevant for some loaders) (default: False)

  • cluster ‚Äì Clustering method to use (one of {'louvain', 'fennel' , 'distributed', 'metis'})

  • num_clusters ‚Äì Target number of clusters for distributed, fennel, or metis.

  • beta ‚Äì Parameter for the distributed clustering algorithm

  • num_iters ‚Äì Maximum iterations for distributed or fennel

  • lr ‚Äì Learning rate

  • cl_model ‚Äì the classification model to use (one of ‚Äúlogistic‚Äù or ‚Äúmlp‚Äù) (default: ‚Äúlogistic‚Äù)

  • cl_train_args ‚Äì extra arguments to pass down to the classification training (default: {})

  • cl_model_args ‚Äì extra arguments to pass to the classfication model constructor (default: {})

  • dist ‚Äì If True, use distance decoder instead of inner product decoder

  • verbose_l2g ‚Äì Verbose output for the alignment step (default: False)

  • normalise ‚Äì If True, normalise the dataset features (default: False)

  • restrict_lcc ‚Äì If True, restrict the dataset to only consider largest connected component (default: False)

  • output ‚Äì output folder

  • levels ‚Äì number of hierarchical patch levels (default: 1)

  • resparsify ‚Äì if > 0, use resistance sparsification for all levels of th hierarchy (default: 0)

  • scale ‚Äì apply scaling transformations during alignment (default: False)

  • rotate ‚Äì apply rotations during alignment (default: True)

  • translate ‚Äì apply translations during alignment (default: True)

  • mmap_edges ‚Äì use memory mapping for edges (only supported by some loaders) (default: False)

  • mmap_features ‚Äì use memory mapping for features (only supported by some loaders) (default: False)

  • random_split ‚Äì use random train-test splits for evaluation (default: True)

  • use_tmp ‚Äì copy data to tmp dir during load (default: False)

  • cluster_init ‚Äì run the cluster initialisation script (default: False)

  • use_gpu_frac ‚Äì fraction of gpu to use by each worker (default: 1.0)

  • grid_search_params ‚Äì use grid search for classification parameters (only for cl_model=‚Äômlp‚Äô)

  • device ‚Äì Device used for training e.g., ‚Äòcpu‚Äô, ‚Äòcuda‚Äô (defaults to ‚Äòcuda‚Äô if available else ‚Äòcpu‚Äô)

  • verbose_train ‚Äì If True, show progress info

  • run_baseline ‚Äì if True, run baseline full model

  • progress_bars ‚Äì show progress bars (default: True)

This function is also exposed as a command-line interface.
